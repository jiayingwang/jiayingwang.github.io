Cryptography

 2020.9.7  2020.9.23  Cryptography  774  4 mins

Cryptography comes from two Greek words "Krypto" and "graphene", in which "Krypto" means hidden, and "graphene" means writing. Cryptography is the art and science of making a cryptosystem that is capable of providing information security. In other words, it studies how to encrypt a message to avoid unauthorized access.

The cryptography provides the following four funamental information security services.

  • Confidentiality: to keep the information from an unauthorized person.
  • Data Integrity: to deteect if data has been manipulated in an unauthorized manner.
  • Authentication: to provide the identification of the originator.
  • Non-repudiation: to ensure that an entity cannot refuse the ownership of a previous commitment or an action.

The following picture shows a cryptosystem, which is also called cipher system. It is an implementation of cryptographic techniques to provide information security services. In the figure, a sender who want to transfer some sensitive data to a receiver in such a way that any inteceptor cannot extract the plaintext.

Components of a Cryptosystem

The components of a cryptosystem are:

  • Plaintext: It is the data to be protected during transmission.
  • Encryption Algorithm: It is a mathematical process to produce a ciphertext for given plaintext and encryption key.
  • Ciphertext: It is encrypted text which will flows on public channel.
  • Decryption Algorithm: It is a mathematical process to produce a plaintext for given ciphertext and decryption key.
  • Encryption Key: It is a value that is known to the sender.
  • Decryption Key: It is a value that is known to the receiver.

For a given cryptosystem, a collection of all possible decryption keys is called a key space.

Types of a Cryptosystem

There are two types of cryptosystems based on the manner of Keys used in the system.

  • Symmetric Key Encryption: The same keys are used for encrypting and decrypting the information. Digital Encryption Standard (DES), Triple-DES (3DES), IDEA, and BLOWFISH are well-known examples of symmetric key encryption methods. The salient features of symmetric key encryption includes:
    1. The sender and receiver must a common key before sending data.
    2. for a group of n users, to establish communication for each pair of users, the number of keys required is $\frac{n\times (n-1)}{2}$.

  • Asymmetric Key Encryption: Different keys are used for encrypting and decrypting the information. Every user in the system needs to have a pair of keys, public key and private key. The salient features of asymmetric key encryption includes:
    1. Each user in this system needs a pair of keys (private key and public key), in which the public key is put in public repository.
    2. for a group of n peers, to establish communication for each pair, the number of keys required is $2n$.

The simplest method of encryption named Caesar Shift Cipher, which encrypts the message by shifting each characters by a given number (e.g. shift each character by 2). While the decrpytion can be achieved by shifting those characters back using the same number. That number (2) will be taken as key. Only those who own the key can decrypt the ciphertext.

For example. Give a plaintext "Hello, nice to meet you!", a sender will encrypt it to a ciphertext Jgnnq."pkeg"vq"oggv"{qw# and send it. For a unauthorized user, it is just a piece of messy code.

H e l l o , n i c e t o m e e t y o u !
J g n n q . " p k e g " v q " o g g v " { q w #

While for an authorized receiver who own the key (2) can easily decrypt it by shifting back for each character. Then he/she will get the original plaintext.

J g n n q . " p k e g " v q " o g g v " { q w #
H e l l o , n i c e t o m e e t y o u !

key = 2

def encrypt(plaintext):
    ciphertext = ''
    for ch in plaintext:
        ciphertext += chr(ord(ch)+key)
    return ciphertext

def decrypt(ciphertext):
    plaintext = ''
    for ch in ciphertext:
        plaintext += chr(ord(ch)-key)
    return plaintext

plaintext = "Hello, nice to meet you!"
print('plainttext:', plaintext)
ciphertext = encrypt(plaintext)
print('after encrypt:', ciphertext)
print('after decrypt:', decrypt(ciphertext))

In the code, ord() function will transform a character to an integer representing the ASCII code of the character. While what does chr() function do is to tranform an integer back into ASCII code.
